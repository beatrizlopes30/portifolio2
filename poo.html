PROGRAMAÇÃO ORIENTADA A OBJETOS COM JAVA: CONCEITOS FUNDAMENTAIS

Java, POO, Programação

A Programação Orientada a Objetos (POO) em Java é um paradigma que organiza o software em torno de objetos, que são instâncias de classes. Esses objetos possuem atributos (dados) e métodos (comportamentos). Java é uma linguagem fortemente baseada em POO, e seus principais conceitos são:

1. Os Quatro Pilares da POO

1. Encapsulamento
  • Esconde os detalhes internos de um objeto, expondo apenas o necessário.
  • Atributos são geralmente privados, e o acesso é controlado por getters e setters.
  • Exemplo: Uma conta bancária não permite alterar o saldo diretamente, apenas por métodos como depositar() e sacar().
2. Herança
  • Permite que uma classe (subclasse) herde atributos e métodos de outra (superclasse).
  • Promove reutilização de código.
  • Exemplo: Carro e Moto podem herdar de Veículo (que tem atributos como marca e ano).
3. Polimorfismo
  • Um objeto pode se comportar de diferentes formas dependendo do contexto.
  • Sobrescrita (Override): Uma subclasse redefine um método da superclasse.
  • Sobrecarga (Overload): Várias versões de um método com parâmetros diferentes.
  • Exemplo: Um método emitirSom() pode ser diferente para Cachorro ("Au au!") e Gato ("Miau!").
4. Abstração
  • Foca no essencial, ignorando detalhes desnecessários.
  • Classes abstratas não podem ser instanciadas, apenas herdadas.
  • Interfaces definem contratos (o que uma classe deve fazer, mas não como).
  • Exemplo: Uma FormaGeometrica pode ter um método calcularArea(), mas cada forma (como Círculo ou Quadrado) implementa de forma diferente.

2. Elementos Principais da POO em Java

Classes e Objetos
  • Classe: Modelo/planta que define atributos e métodos.
  • Objeto: Instância concreta de uma classe (ex: Carro meuCarro = new Carro()).
Construtores
  • Métodos especiais que inicializam objetos quando são criados.
  • Podem ter diferentes versões (sobrecarga de construtores).
Modificadores de Acesso
  • public: Acesso livre.
  • private: Acesso apenas dentro da classe.
  • protected: Acesso dentro do pacote e subclasses.
  • (Sem modificador): Acesso apenas dentro do pacote.
Getters e Setters
  • Padrão para controlar acesso a atributos privados.
  • Exemplo: getNome() retorna o nome, setNome(String nome) altera o nome.
Métodos Estáticos
  • Pertencem à classe, não ao objeto (ex: Math.sqrt()).

3. Conceitos Avançados

Interfaces
  • Definem o que uma classe deve fazer, mas não como.
  • Uma classe pode implementar várias interfaces.
  • Exemplo: Autenticavel pode exigir um método autenticar().
Classes Abstratas
  • Não podem ser instanciadas, apenas herdadas.
  • Podem ter métodos abstratos (sem implementação) e concretos.
Tratamento de Exceções
  • try-catch-finally lida com erros de forma controlada.
Classes e Métodos final
  • final em classe: Não pode ser herdada (ex: String).
  • final em método: Não pode ser sobrescrito.

4. Boas Práticas em POO com Java

  • Princípio da Responsabilidade Única: Cada classe deve ter apenas uma função.
  • Encapsulamento Forte: Atributos privados, acesso controlado.
  • Prefira Composição em vez de Herança: Mais flexível e evita hierarquias complexas.
  • Use Interfaces para Contratos: Melhor que herança múltipla.
  • Documente com Javadoc: Facilita a manutenção.

5. Exemplo Prático (em palavras)

Imagine um sistema bancário:

  • Classe ContaBancaria (atributos: saldo, titular; métodos: depositar(), sacar()).
  • Subclasse ContaCorrente (herda ContaBancaria e adiciona limiteChequeEspecial).
  • Interface Tributavel (exige método calcularImposto()).
  • Polimorfismo: Um objeto ContaBancaria pode ser tanto ContaCorrente quanto ContaPoupanca.

A POO em Java ajuda a criar sistemas modulares, reutilizáveis e de fácil manutenção. Dominar encapsulamento, herança, polimorfismo e abstração é essencial para desenvolver software robusto e escalável.
